# セキュリティポリシー / Security Policy

## このプロジェクトの位置づけ（重要）

本リポジトリは、静岡市清水区 三保〜清水港を題材とした **平時の防災学習・避難訓練支援用の試作（大学ゼミの研究）** です。
**実際の災害時の避難誘導に使用するためのものではありません。** 実災害時は、自治体・気象庁・消防・警察等の
**公式情報に必ず従ってください。** 本アプリの表示（避難先・経路・距離・伝承・記録高など）は学習用の参考であり、
安全・危険を保証するものではありません。

## 脆弱性の報告方法

セキュリティ上の問題（依存パッケージの脆弱性、XSS、情報漏えい等）を見つけた場合は、
**公開 Issue を作成せず**、以下のいずれかで非公開に報告してください。

1. **GitHub の Private vulnerability reporting**（推奨）
   リポジトリの **Security** タブ →「Report a vulnerability」から報告できます
   （メンテナが Settings → Security → "Private vulnerability reporting" を有効化している必要があります）。
2. それが使えない場合は、リポジトリオーナー（`@towa1824`）に GitHub 経由で連絡してください。

報告には、再現手順・影響範囲・該当ファイル/行・想定される深刻度を含めてください。

## 対応について

本プロジェクトは研究・学習目的のため、対応はベストエフォートです。
受領後、内容を確認し、妥当なものは修正のうえ必要に応じて GitHub Security Advisory で公表します。

## 対象範囲

- 対象: 本リポジトリのコード・設定（`src/`, `tools/`, ビルド設定, GitHub Actions）。
- 対象外: 外部サービス（OpenStreetMap タイル、Overpass API、OSRM、国土地理院 API、Nominatim 等）そのものの脆弱性。
  これらは各サービスの提供元へご連絡ください。

## 既知の前提（脆弱性ではない事項）

- 開発サーバの HTTPS は `@vitejs/plugin-basic-ssl` による **自己署名証明書** です（ローカル開発用）。
- 経路・距離は OSRM のデモサーバ（`routing.openstreetmap.de`）由来の **参考表示** で、徒歩最短を保証しません。
- 一部の避難施設・伝承は座標が未確定（`lat/lng=null`）で、地図/AR に表示されません（`data_quality.md` 参照）。
