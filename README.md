# 三保〜清水港 津波避難 × 災害伝承 AR（学習用試作）

静岡市清水区の **三保〜清水港周辺** を対象に、**津波避難施設**（津波避難タワー・命山・津波避難ビル）と
**地域の災害伝承・史料**（御穂神社の津波痕跡、安政東海地震の寺院避難記録、瀬織戸の渡し など）を
ひとつの地図／AR で結びつけて学べる、防災学習・避難訓練支援用の Web アプリです。

> **⚠ このシステムは平時の防災学習・避難訓練支援を目的とした試作です。
> 実際の災害時は自治体・気象庁・消防・警察等の公式情報に従って避難してください。**

---

## 研究上の新規性（既存サービスとの差分）

本システムは、単に現在地から避難先への経路や AR 矢印を表示するだけでなく、
静岡市清水区三保〜清水港周辺に残る災害伝承・史料情報を、現在の津波避難施設情報と
結びつけて提示する。これにより、利用者が **「どこへ逃げるか」だけでなく、
「なぜその方向へ避難するのか」** を理解できる防災学習支援を目指す。

具体的には、一般的な避難ナビが「最寄りの避難所＋矢印」を出すのに対し、本システムは:

- 各避難先カードに **「なぜここへ逃げるか」** の理由（過去の津波記録に基づく短文）を必ず添える
- AR/地図に、その場所の **災害伝承・史料**（安政東海地震の被害記録・避難記録）を重ねる
- 史料に津波高の記録がある地点では、AR 空間に **「記録高ゲージ」（高さの柱）** を立て、
  *どこまで水が来たと記録されているか* を直感的に体感できるようにする
- 寺院の被害記録（行谷 2006）から推定された **震度** を史料地点に表示し、
  「揺れ → 津波」という時間順序の避難行動を学べるようにする

伝承・史料は **避難判断の決定根拠ではなく、公的な津波避難施設情報と組み合わせて理解するための
補助情報** として扱い、「伝承だけで特定の場所・経路の安全/危険を断定しない」ことを UI 全体で徹底する。

## 起動方法

### 必要環境
- Node.js 18 以上（推奨 20+）
- モバイル実機で AR（カメラ・方位センサー）を使う場合は HTTPS が必須

### セットアップ

```bash
cd miho-tsunami-densho-ar
npm install
```

### 開発サーバ起動

```bash
# 既定: HTTPS で起動（スマホ実機のAR=カメラ/方位センサー用）
npm run dev
# → https://localhost:5173/  （自己署名証明書の警告は「詳細 → アクセスする」で続行）
# → 同じWi-Fiのスマホからは https://<PCのIPアドレス>:5173/ にアクセス

# デスクトップでのデモモード確認だけなら HTTP でも可（カメラは使わない）
npm run dev:http
# → http://localhost:5173/
```

> **なぜ HTTPS か**: `getUserMedia`（カメラ）/ `Geolocation`（GPS）/ `DeviceOrientation`（方位）は
> ブラウザの仕様上 HTTPS（セキュアコンテキスト）でしか動きません。本リポジトリは
> `@vitejs/plugin-basic-ssl` で自己署名証明書を自動生成します。

### ビルド

```bash
npm run build         # dist/ に静的ファイルを出力
npm run preview       # ビルド結果をローカル確認
npm run validate:data # data/*.json の健全性チェック（id重複・出典・座標・location_kind）
```

### 緯度経度の補完（任意・データ更新時のみ）

```bash
npm run geocode    # data/*.json の lat/lng=null を GSI/Nominatim で補完（要 Python 3）
```

### 公開（GitHub Pages・自動デプロイ）

`main` に push すると `.github/workflows/deploy-pages.yml` がビルドして GitHub Pages へ自動デプロイします。

- 公開URL: **https://towa1824.github.io/miho-tsunami-densho-ar/**
- 初回のみ設定が必要: **Settings → Pages → Build and deployment → Source** を「**GitHub Actions**」にする
- Pages は HTTPS 配信なので、公開URLでは GPS・カメラ・方位センサー（セキュアコンテキスト必須）も動く
- 公開されるのは `dist/`（ビルド成果物）のみ

## 使い方

1. 画面上部の **「現在地」** で現在地を決める。3通り:
   - **地図を直接クリック** → クリックした地点が現在地になる（三保〜清水港の好きな場所からの避難案内を試せる）
   - **デモ地点**（三保松原 / 御穂神社 / 清水港・日の出 / 専念寺）を選ぶ
   - **GPSを使う**（屋外・HTTPS）
2. **「避難施設」タブ**: 現在地から近い津波からの一時避難先 上位3件＋指定避難所（区別表示）。
   各施設まで距離と所要時間（上部バーの **🚶徒歩 / 🚗車** で切替）を表示。「地図で経路」で道路経路・
   進行方向矢印・距離/所要時間サマリーを表示する。
   候補は可能なら **道路距離（参考経路）順** に並べ替え、道路距離が取得できない時は **直線距離順**
   （カードに「道路距離／直線距離」を明記）。**経路・距離はすべて参考表示であり、徒歩最短ルートを保証しない**。
   施設名・住所からの推定座標は「📍 おおよその位置」と表示する
3. **「伝承・史料」タブ**: 近い順に災害伝承・史料を表示。記録上の津波高・推定震度も表示
4. **「現地目線」タブ**: 選択した避難先への方向カード（コンパス）＋一人称ビュー。2つのモードを切替できる:
   - **🧭 現地目線ビュー（おすすめ・既定）**: その場に立った目線の3D表示。カメラも方位センサーも不要で
     **HTTPのローカル表示でも動く**。**実際の町並み（建物・道路・松原・砂浜・海岸線）を
     OpenStreetMap から取得して立体表示**（建物は height / building:levels タグで押し出し。
     タグの無い建物は種別ごとの既定高さによる概形）。**建物・道路・公園の名前**（OSM `name`）も
     ラベル表示する。その上に避難施設を色つきの建物、伝承を看板柱として実寸配置する。
     **避難経路は開いた時点で必ず OSRM 道路経路を取得**し、その道なりに沿って細い半透明の青ライン＋
     白シェブロンで描く（経路が取れない時は青ルートを出さず、灰色破線で「目的地方向＝直線参考」とだけ示す）。
     案内は最終目的地ではなく**「次に進む経路セグメント」基準**: 初期表示は経路の最初の進行方向を正面にし、
     HUD・ガイド矢印・ミニマップ（実経路ポリライン）・下部ナビカードが「次は◯◯へ約N m／のこり約M m」を示す。
     距離・所要時間は**道路経路ベース**（OSM/OSRM 由来の参考表示。徒歩避難の正式経路ではない旨を明記）。
     下部の説明カードは**折りたたみ式**（既定は1行サマリ、タップで詳細）。
     **操作は Google ストリートビュー風**: ドラッグで見回し（離すと慣性で少し流れる）、
     行きたい場所を**タップ/クリックするとその地点へ前進**。矢印キー/WASD（画面の◀▶▲▼）でも歩ける。
     地図の施設ポップアップ「🧭 現地目線で見る」からも直接開ける。
   - **📷 ARカメラ（現地・上級）**: スマホ実機のカメラ映像に矢印・説明・伝承・記録高ゲージを
     実方位で重ねる（**HTTPS＋カメラ・方位センサー許可が必要**、屋外推奨）。センサーが無い端末では
     自動で現地目線ビュー相当の操作に切り替わる。
5. **「注意事項」タブ**: 必須注意文・研究上の新規性・使用資料一覧・位置未取得項目

### 現地に行かずに確認する（開発・発表のコツ）
方位センサーの無い PC でも、**現地目線ビュー**で三保〜清水港の配置を再現・確認できる。
PDFガイドの「fakeGps で開発 → 屋外で検証」の二段構えに沿った設計で、
1. 「現在地」でデモ地点（三保松原など）を選ぶ → 擬似GPSでその場所に立った状態を再現
2. 「現地目線」→「現地目線ビューを開く」→ ドラッグで見回す／歩く／「避難先を正面に」
3. 矢印・伝承ボード・記録高ゲージの方位・距離・高さが正しいかを画面上で確認できる

## データ構成

| ファイル | 内容 |
|----------|------|
| `data/evacuation_facilities.json` | 津波避難タワー・命山・津波避難ビル・（区別用に）指定避難所 |
| `data/tradition_points.json` | 災害伝承・史料ポイント（津波痕跡・寺院避難記録・瀬織戸の渡し 等） |
| `data/additional_shimizu_disaster_traditions.json` | 追加の災害伝承・史料（清水区の寺院被害記録、海長寺の記録継承、巴川流域の水害記憶、河内の大石 等）。`src/data.js` で `tradition_points.json` とマージし、伝承・史料タブ／地図／現地目線ビュー／ARから1つの `traditions` として参照する |
| `data/sources.json` | 使用資料の台帳（全データに `source_url`/`source_title` を付与） |
| `data/demo_locations.json` | デモ現在地（発表・訓練用） |

各レコードは `confidence`（high/medium/low）を持ち、緯度経度が不確かなものは medium 以下にしている。
**出典のない情報は採用していない。推測で緯度経度や史料内容を作っていない。**

> `confidence` は **出典・存在の信頼度** を表す軸で、**座標の精度とは別**である。
> 公式資料に存在が明記されていても座標が確定できない施設（三保のタワー・命山など）は
> `confidence=high` のまま `lat/lng=null`（地図/AR非表示）とし、その理由を `geocode_note` に明記している。
> `npm run validate:data` はこれを **警告** として可視化する（理由が未記載ならエラー）。

### 伝承・史料の地点種別（`location_kind`）

伝承・史料の各レコードは、地図/AR で **「点として断定してよいか」** を区別する `location_kind` を持つ。

| location_kind | 意味 | 地図/ARでの扱い |
|---------------|------|----------------|
| `exact_point` | 寺社・施設など点として扱える | 通常のマーカー／看板柱として表示 |
| `representative_point` | 旧地名・小字・範囲の代表点（吹合・江湖・向島・瀬織戸 等） | 表示するが「📍 代表点（おおよその位置）」と明記 |
| `area_or_line` | 本来は面/ライン（巴川流域の水害記憶など） | 便宜的に代表点表示するが「📍 代表点・本来は面/ライン情報」と明記 |
| `unresolved` | 座標なし（河内の大石 等） | 地図/AR には出さず、注意事項タブの「位置未取得一覧」に表示 |

代表点・面情報は、ポップアップ・カード・現地目線ビューの解説に注記を出し、**特定地点の被害そのものと誤解されないように**している。
AR（現地目線ビュー）は原則 `exact_point` を主対象とし、`representative_point`/`area_or_line` は注記つきで扱う。

## 技術構成

- **WebAR**: LocAR.js（位置情報AR）+ three.js（描画）+ Vite — PDFガイドの推奨構成に準拠
- **地図**: Leaflet + OpenStreetMap タイル（APIキー不要）
- **町並み3D**: OpenStreetMap（Overpass API・無料/APIキー不要）から建物・道路・水域・緑地・海岸線を取得し
  three.js で立体化。localStorage に7日キャッシュ。© OpenStreetMap contributors（ODbL）
- **経路**: OSRM（routing.openstreetmap.de・APIキー不要のデモサーバ）。徒歩=foot プロファイル（歩行者が通行可能な道路ネットワーク）、車=car プロファイル（車道）で道筋を出し分ける。距離・所要時間は OSRM の道路距離/duration を使う。**いずれも「参考経路」であり徒歩最短ルートを保証しない**。避難施設タブでは現在地確定時に候補へ道路距離を引いて並べ替える（取得できなければ直線距離順）。OSRM 不達時は灰色破線の「直線参考」にフォールバックし、その旨を明示する
- カメラ映像は自前の `<video>` 背景で管理し、LocAR は GPS 投影・方位センサーに使用

## 禁止事項（このプロジェクトで守っていること）

- 伝承だけを根拠に「この道は危険」「この場所は安全」と断定しない
- 実災害時に使える本番避難誘導アプリであるかのように書かない（あくまで学習・訓練支援の試作）
- 出典のない史料・伝承を作らない／推測で緯度経度や史料内容を作らない
- 公式情報と矛盾する避難案内をしない
- 緯度経度が不確かな場合は `confidence` を medium または low にする
